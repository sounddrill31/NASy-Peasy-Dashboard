How It Works

1. Add a Shared Directory (a folder accessible on your host, e.g. /app/your-data-folder).
2. Create a Virtual Volume under it — this creates a subdirectory (e.g. /app/your-data-folder/DATA).
3. In your app's docker-compose.yaml, reference it as $DATA — the path is substituted automatically at deploy time.
4. A red dot means the directory no longer exists on disk (non-blocking warning).
